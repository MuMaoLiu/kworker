# 03 - 架构设计

## 1. 四级信任链

```
┌─────────────────────────────────────────────────────────┐
│ [L0] RK3568 BootROM（OTP 公钥哈希，一次性 fuse）         │
│      ↓ 验签 MiniLoaderAll / SPL                         │
├─────────────────────────────────────────────────────────┤
│ [L1] U-Boot FIT 签名（FIT_HW_CRYPTO + RSA2048/4096）    │
│      ↓ 验签 U-Boot 自身                                  │
├─────────────────────────────────────────────────────────┤
│ [L2] HVB 链式校验                                        │
│      RVT → boot_linux / resource / ramdisk /            │
│            system / vendor / sys-prod / chip-prod       │
│      ↓ 写 bootargs（ohos.boot.hvb.*）                   │
├─────────────────────────────────────────────────────────┤
│ [L3] Kernel dm-verity                                    │
│      init 按 fstab hvb 标志挂载只读校验分区              │
│      ↓                                                   │
├─────────────────────────────────────────────────────────┤
│ [L4] 运行时                                               │
│      fscrypt（userdata 加密）                            │
│      fs-verity（可选，关键文件完整性）                    │
└─────────────────────────────────────────────────────────┘
```

任何一级校验失败 → 拒绝启动 / 强制进入 rockusb 恢复模式。

## 2. HVB 验签数据流

```
编译时                              启动时
──────                              ──────
hvbtool / kh_hvbtool                U-Boot bootkhp
  │                                   │
  ├─ 计算镜像 hash/hashtree           ├─ 读 RVT 分区
  ├─ 生成 HVB Cert                    ├─ 验签 RVT（PRK 公钥）
  ├─ 签名写入 Footer                  ├─ 链式验签各分区 Footer
  └─ 写入 RVT 公钥列表                ├─ 检查 rollback_index
                                      ├─ 提交 RPMB（仅 lock 态）
                                      └─ 设置 bootargs → Kernel
                                              │
                                          init 读 bootargs
                                              │
                                          dm-verity 挂载 system 等
```

## 3. HVB 核心数据结构

### 3.1 分区 Footer（每个参与验签的镜像末尾）

| 字段 | 说明 |
|------|------|
| signature_info | 签名内容（可含多份 pubkey，兼容旧 bootloader） |
| hash_info | 完整性信息（hash 或 hashtree） |

### 3.2 RVT（Root Verity Table）分区

统一存放各镜像 verity 信息的公钥列表。Bootloader 只需校验 RVT 合法性，即可用 RVT 内 pubkey 逐个校验各分区。

### 3.3 校验类型

| 类型 | 适用场景 | 说明 |
|------|----------|------|
| **hash** | boot_linux、ramdisk、resource | 整包 hash，启动时一次性校验 |
| **hashtree** | system、vendor、sys-prod | 按块校验，依赖 dm-verity，不拖慢开机 |

## 4. bootargs 参数（U-Boot → Kernel）

| 参数 | 说明 |
|------|------|
| `ohos.boot.hvb.enable` | `true` / `false` / `green`/`yellow`/`red`/`orange` |
| `ohos.boot.hvb.version` | 当前 1.0 |
| `ohos.boot.hvb.hash_algo` | sha256 |
| `ohos.boot.hvb.digest` | 所有受保护镜像 + RVT 的 verity 摘要 |

init 根据上述参数和 fstab 中 `hvb` 标志启用 dm-verity。

## 5. 与 Android AVB 的关系

当前 U-Boot 同时编译了 AVB 和 HVB 库。**建议策略**：

| 方面 | 决策 |
|------|------|
| 主启动路径 | `bootkhp`（HVB 校验后加载 extlinux） |
| system 分区校验 | HVB + dm-verity（非 AVB vbmeta） |
| AVB 保留用途 | fastboot 兼容 / OTA 过渡（可选） |
| 长期 | 统一 HVB，减少双栈维护 |

启动命令选择逻辑（`rockchip-common.h`）：

- `CONFIG_FIT_SIGNATURE=y` → `bootkhp`
- `CONFIG_AVB_VBMETA_PUBLIC_KEY_VALIDATE` → `boot_android`

**推荐**：开启 `CONFIG_FIT_SIGNATURE` + `CONFIG_HVB_LIBHVB`，走 bootkhp 路径。

## 6. 模块划分

```
extension/khsl/ohvb/              # Kaihong 封装层（建议新建）
├── tools/
│   ├── provision.sh              # 产线：密钥→OTP→首次签名→lock
│   ├── status.sh                 # 查看 lock/rollback/验签状态
│   └── ota_resign.sh             # OTA 后重签名
├── docs/                         # 本文档目录

base/startup/hvb/                 # 复用官方 libhvb + hvbtool
├── libhvb/                       # 验签库
└── tools/
    ├── hvbtool.py
    ├── kh_hvbtool.py             # Kaihong 扩展
    └── hvb_sign.py               # 构建签名

device/board/kaihong/khdvk_rk3568_a/
├── uboot/configs/                # HVB Kconfig 开关
├── loader/parameter.txt          # 增加 rvt 分区
└── kernel/config/                # dm-verity 内核宏

vendor/kaihong/khdvk_rk3568_a/
└── startup/init/cfg/fstab.rk3568 # hvb 标志

device/soc/rockchip/tools/uboot/rk356x/
├── cmd/bootkhp.c                 # 启动 + hvb_verify
└── lib/hvb/                      # U-Boot HVB 实现
```

## 7. 参与 HVB 签名的镜像列表

来自 `hvb_sign.py`：

```
rvt, boot_linux, resource, ramdisk, system, vendor, sys_prod, chip_prod
```

## 8. 密钥体系（Kaihong kh_hvbtool）

| 密钥 | 用途 |
|------|------|
| **PRK**（Platform Root Key） | 根信任，验签 RVT |
| **PSK**（Platform Signing Key） | 签名各分区 footer |
| **secure_boot key** | 签名 permanent_attributes |
| **product_id** | 设备产品标识 |
| **salt** | 永久属性盐值 |

产线流程参见 [06-运维与管理.md](./06-运维与管理.md)。
