# KHSL (Kaihong System Linux) 功能验证指南

本文档记录了在 KaihongOS (RK3568) 上验证 KHSL 核心功能（微内核隔离、资源限制、多用户管理、TTY 交互）的完整步骤。

**前提条件：** 请确保已经烧录了包含最新 `kh_term_daemon`（已恢复为系统守护进程模式）和 `kh_term_daemon.cfg` 的系统镜像。

---

## 阶段一：环境准备与初始化

如果刷机后 `/data` 分区被清空，请执行此阶段重新部署 Ubuntu 根文件系统。如果 `/data` 数据还在，可直接跳到阶段二。

**1. 推送镜像与初始化脚本**
在 Windows PowerShell 中执行（请根据实际路径调整）：
```powershell
hdc file send \\wsl.localhost\master\home\liuboyi\master\openharmony\extension\khsl\distribution\khsl-ubuntu-22.04-arm64.tar.gz /data
hdc file send \\wsl.localhost\master\home\liuboyi\master\openharmony\extension\khsl\distribution\khsl_init_ubuntu.sh /data
```

**2. 执行初始化脚本**
```powershell
hdc shell "cd /data && chmod +x khsl_init_ubuntu.sh && ./khsl_init_ubuntu.sh"
```
*(等待脚本自动完成虚拟磁盘创建、格式化、挂载、解压，以及配置国内源并安装基础工具，直到输出 `[KHSL] Initialization completed!`)*

---

## 阶段二：验证守护进程与微内核隔离

**1. 确认守护进程已在后台运行**
```powershell
hdc shell "ps -ef | grep kh_term_daemon"
```
*(预期：能看到 `root ... kh_term_daemon` 进程，说明开机自启配置生效)*

**2. 使用客户端进入 KHSL 环境**
```powershell
hdc shell khsl
```
*(预期：提示符变为 `root@localhost:/#`。注意：这里使用的是 `khsl` 客户端，而不是直接运行 daemon)*

**3. 验证文件系统隔离 (Chroot)**
在 KHSL shell 中执行：
```bash
cat /etc/os-release
```
*(预期：输出 `PRETTY_NAME="Ubuntu 22.04.5 LTS"` 等 Ubuntu 系统信息)*
```bash
ls -la /
```
*(预期：看到纯净的 Ubuntu 根目录结构，如 bin, boot, dev, etc, home 等，没有 KaihongOS 的 system/vendor)*

**4. 验证 Cgroups 资源限制**
```bash
cat /sys/fs/cgroup/memory/khsl/memory.limit_in_bytes
```
*(预期：输出 `209715200`，即 200MB 内存限制生效)*

---

## 阶段三：验证 TTY 交互体验

在 KHSL shell 中尝试以下操作：
1. **方向键**：按上下左右键，预期光标正常移动或翻阅历史命令，不再出现 `^[[A` 等乱码。
2. **Tab 补全**：输入 `his` 按 Tab 键，预期能自动补全为 `history`。
3. **信号处理**：输入 `sleep 10` 回车，然后按 `Ctrl+C`，预期能中断 sleep 命令，且**不会**退出 KHSL 环境。

---

## 阶段四：验证 WSL 风格的多用户管理

**1. 首次进入触发注册流程**
在清理了 `/data/khsl/rootfs/var/lib/khsl_default_user` 的情况下，执行：
```powershell
hdc shell khsl
```
*(预期：提示 `Installing, this may take a few minutes...`，并要求输入用户名和密码两次。)*

**2. 验证默认用户切换**
注册成功后，提示符会变成 `username@localhost:~$`。
执行验证命令：
```bash
whoami
# 预期输出：您刚才注册的用户名

sudo ls /root
# 预期：提示输入密码，输入密码后能列出 root 目录内容
```

**3. 验证再次进入免注册**
输入 `exit` 退出 KHSL。
再次执行 `hdc shell khsl`。
*(预期：直接进入 `username@localhost:~$`，不再提示注册。)*

**4. 验证多开并发**
保持当前的 PowerShell 窗口不退出。
**新开一个 Windows PowerShell 窗口**，执行：
```powershell
hdc shell khsl
```
*(预期：第二个窗口也能瞬间进入 `username@localhost:~$`，两个窗口独立操作，互不干扰)*
